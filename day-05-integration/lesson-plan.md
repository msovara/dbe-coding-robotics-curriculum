# Day 5: Integration, Assessment, and Capstone Project
## Detailed Lesson Plan

**Duration:** 6-8 hours (Full Day)
**Target Audience:** DBE Teachers (Mixed Experience Levels)
**Prerequisites:** Days 1-4 completion
**Learning Objectives:**
- Integrate coding/robotics into DBE curriculum
- Develop assessment strategies
- Create comprehensive lesson plans
- Complete capstone project
- Plan implementation in schools

---

## Morning Session (3-4 hours)

### Session 1: Curriculum Integration Workshop (2 hours)
**Time:** 08:00 - 10:00 (with 15-minute break at 09:00)

#### Learning Objectives
- Map coding/robotics to DBE subjects
- Create integration plans
- Design cross-curricular projects
- Address curriculum requirements

#### Activities

**Opening (10 minutes)**
- Welcome and energy check
- Review of Days 1-4
- Introduction: "Today we put it all together!"
- Set goals for the day

**Curriculum Mapping Activity (30 minutes)**
- **DBE Subjects Analysis:**
  - Mathematics
  - Natural Sciences
  - Technology
  - Languages
  - Life Skills
  - Social Studies

- **Activity:** "Subject Integration Brainstorm"
  - In groups by subject area
  - List topics from your curriculum
  - Identify where coding/robotics fits
  - Share with whole group

**Integration Examples (30 minutes)**
- **Mathematics:**
  - Problem-solving with Scratch
  - Data collection and graphing
  - Patterns and sequences
  - Geometry and coordinates

- **Natural Sciences:**
  - Data collection experiments
  - Environmental monitoring
  - Scientific method
  - Measurement and observation

- **Technology:**
  - Systems thinking
  - Design process
  - Automation
  - Innovation

- **Languages:**
  - Interactive storytelling
  - Communication projects
  - Presentation skills
  - Digital literacy

**Break (15 minutes)**

**Creating Integration Plans (45 minutes)**
- **Template Provided** (see resources)
- **Components:**
  1. Subject and grade level
  2. Curriculum topic
  3. Learning objectives
  4. Coding/robotics activity
  5. Assessment methods
  6. Resources needed
  7. Timeline

- **Activity:** Create Your Integration Plan
  - Choose your subject/grade
  - Select a topic
  - Design coding/robotics activity
  - Plan assessment
  - Share with partner for feedback

**Cross-Curricular Projects (20 minutes)**
- **Examples:**
  - Environmental monitoring (Science + Math + Technology)
  - Interactive story (Languages + Technology)
  - Data collection project (Math + Science)

- **Activity:** Design cross-curricular project
  - Identify multiple subjects
  - Plan integrated activity
  - Consider assessment across subjects

### Session 2: Assessment Strategies (1-2 hours)
**Time:** 10:15 - 12:00

#### Learning Objectives
- Understand formative vs. summative assessment
- Create rubrics for coding projects
- Design portfolio-based assessment
- Plan peer and self-assessment

#### Activities

**Assessment Types (20 minutes)**
- **Formative Assessment:**
  - Ongoing, during learning
  - Quick checks, observations
  - Guides instruction
  - Low-stakes

- **Summative Assessment:**
  - End of unit/project
  - Evaluates learning
  - Higher stakes
  - Demonstrates mastery

- **Discussion:** When to use each type?

**Rubrics for Coding Projects (30 minutes)**
- **Components of Good Rubric:**
  - Clear criteria
  - Performance levels
  - Descriptors for each level
  - Student-friendly language

- **Sample Rubric Review:**
  - Technical skills
  - Creativity
  - Problem-solving
  - Collaboration
  - Documentation

- **Activity:** Create Your Rubric
  - Choose a project
  - Identify assessment criteria
  - Define performance levels
  - Write descriptors

**Portfolio Assessment (20 minutes)**
- **What is Portfolio Assessment?**
  - Collection of student work
  - Shows growth over time
  - Multiple projects
  - Reflection included

- **Portfolio Components:**
  - Projects (Scratch, Arduino)
  - Documentation
  - Reflections
  - Peer feedback
  - Self-assessments

- **Activity:** Plan portfolio structure
  - What to include?
  - How to organize?
  - How to assess?

**Peer and Self-Assessment (20 minutes)**
- **Peer Assessment:**
  - Students review each other's work
  - Provides feedback
  - Develops critical thinking
  - Builds community

- **Self-Assessment:**
  - Students evaluate own work
  - Promotes reflection
  - Develops metacognition
  - Ownership of learning

- **Activity:** Practice peer assessment
  - Review partner's project
  - Provide constructive feedback
  - Use assessment criteria

**Lunch Break (12:00 - 13:00)**

## Afternoon Session (3-4 hours)

### Session 3: Capstone Project (2-3 hours)
**Time:** 13:00 - 15:30

#### Learning Objectives
- Design complete project
- Integrate all learned concepts
- Build and test project
- Present to peers

#### Activities

**Project Requirements (15 minutes)**
- **Must Include:**
  - Scratch programming
  - Arduino hardware
  - Cloud/data element
  - Subject integration
  - Documentation

- **Project Ideas:**
  1. Smart classroom monitor
  2. Interactive science experiment
  3. Educational game with physical controls
  4. Environmental data logger
  5. Subject-specific project

**Planning Phase (30 minutes)**
- **Design Document:**
  - Project goal
  - Components needed
  - Circuit diagram
  - Code outline
  - Integration with curriculum
  - Assessment plan

- **Activity:** Create project plan
  - Choose project
  - Design system
  - List components
  - Plan implementation

**Building Phase (60 minutes)**
- **Work on Projects:**
  - Build circuits
  - Write code
  - Test and debug
  - Document process

- **Support:**
  - Facilitators available
  - Peer support
  - Troubleshooting help
  - Resource sharing

**Break (15 minutes)**

**Presentation Preparation (20 minutes)**
- **Presentation Structure:**
  - Project overview
  - Demonstration
  - Challenges faced
  - Learning outcomes
  - Classroom application

- **Activity:** Prepare 3-minute presentation

**Project Presentations (40 minutes)**
- **Format:** 3 minutes per project
  - 2 minutes: Presentation
  - 1 minute: Questions

- **Feedback:**
  - Peer feedback forms
  - Facilitator feedback
  - Celebration of achievements

## Assessment

### Capstone Project Rubric

| Component | Excellent (4) | Good (3) | Developing (2) | Needs Work (1) |
|-----------|-------------|----------|---------------|----------------|
| **Integration** | All concepts integrated seamlessly | Most concepts integrated | Some concepts integrated | Few concepts integrated |
| **Functionality** | Works perfectly, exceeds requirements | Works as intended | Works with minor issues | Doesn't work or major issues |
| **Creativity** | Highly creative and innovative | Creative solution | Some creative elements | Basic implementation |
| **Documentation** | Excellent, comprehensive | Good documentation | Basic documentation | Minimal documentation |
| **Presentation** | Clear, engaging, well-prepared | Clear and prepared | Adequate presentation | Unclear or unprepared |
| **Curriculum Link** | Strong, clear connection | Good connection | Some connection | Weak connection |
